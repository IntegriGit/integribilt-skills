# api-design-principles

## What I do
I master REST and GraphQL API design principles to help you build intuitive, scalable, and maintainable APIs. I provide guidance on defining consumers, modeling resources, specifying errors, versioning, pagination, auth strategy, and validating with examples.

## How to use me
Use me when designing new REST or GraphQL APIs, refactoring existing APIs, establishing design standards, reviewing API specs, migrating between paradigms, or creating developer-friendly documentation. Do not use me for infrastructure-only work without API contracts or if you only need implementation guidance for a specific framework.
